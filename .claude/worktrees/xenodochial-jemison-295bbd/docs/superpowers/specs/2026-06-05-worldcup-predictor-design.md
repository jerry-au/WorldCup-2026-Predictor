# 世界杯球队对战获胜几率预测 App — 产品功能设计文档

> 版本：v1.1 | 日期：2026-06-09 | 状态：设计阶段

## 一、产品定位

一款基于 **2026 美加墨世界杯（48 队）** 的球队对战获胜几率预测 App，支持 **单场预测**、**完整赛事模拟** 及 **投注推荐分析**，帮助球迷量化分析比赛走向并发现投注价值机会。

---

## 四、概率算法设计

### 4.1 混合泊松-Elo 模型（推荐方案）

参考博彩行业标准模型（Dixon & Coles, 1998），结合 Elo 评分系统解决国际比赛数据稀疏问题，采用四阶段分层架构：

```
Stage 1: 基础概率（Elo → 期望进球）
  综合评分 = 0.6 × Elo + 0.3 × FIFA + 0.1 × 身价归一化
  ↓ logistic 转换
  Elo 差异 → 期望进球（Poisson λ）

Stage 2: 上下文调整（乘性修正因子）
  × 近期状态漂移（指数加权移动平均, λ=0.85）
  × 大赛经验（阵容平均世界杯出场数）
  × 教练系数（顶级教练+5%）
  ↓
  调整后的 λ_A, λ_B

Stage 3: Dixon-Coles 双变量泊松
  P(A进x球, B进y球) = 修正泊松（含平局关联参数 ρ）
  ↓
  累加 → P(胜), P(平), P(负), 各比分概率

Stage 4: 实时修正因子
  × 核心伤病（调减2-8%）
  × 赔率校正（与市场赔率差距大时部分融合）
```

### 4.2 核心公式

#### Elo 期望进球映射

```python
λ_A = μ × e^(δ × (Elo_A - Elo_B) / 400)
λ_B = μ / e^(δ × (Elo_A - Elo_B) / 400)
```

- μ ≈ 2.5（世界杯场均进球常数）
- δ = 校准斜率参数（历史数据拟合得出）

#### 双变量泊松（Dixon-Coles 修正）

```
P(x, y) = τ(x, y) × (λ_A^x × e^-λ_A / x!) × (λ_B^y × e^-λ_B / y!)

τ(x, y) = { 1 - ρ × λ_A × λ_B,          x = y = 0
           { 1 + ρ × λ_A,                x = 0, y = 1
           { 1 + ρ × λ_B,                x = 1, y = 0
           { 1,                           其他

P(胜) = Σ_{x>y} P(x, y)
P(平) = Σ_{x=y} P(x, y)
P(负) = Σ_{x<y} P(x, y)
```

### 4.3 与原模型的区别

| 维度 | 原模型（四维度加权） | 混合泊松-Elo 模型 |
|------|-------------------|-------------------|
| 权重基础 | 主观赋值 35/25/25/15 | Elo 系统自带校准，60+ 年数据支撑 |
| 因子关系 | 简单加性 | 分阶段乘性修正，符合实际 |
| 平局概率 | "基于实力差独立建模"模糊表述 | 双变量泊松自然得出 |
| 实时因素 | 固定 15% 独立维度 | 作为修正因子，重要比赛影响大，悬殊比赛影响小 |
| 比分预测 | 不支持 | 自然产出精确比分概率 |
| 数据稀疏 | 各维度依赖不同来源，数据不一致 | Elo 跨洲际可比，解决国家队数据稀疏问题 |
| 行业对标 | 自定义加权 | 参考 Dixon-Coles 行业标准模型 |

### 4.4 赛事模拟 — 蒙特卡洛方法

1. 根据混合泊松模型生成单场比分概率分布
2. 随机抽样模拟比赛结果
3. 按赛制（小组赛 → 32 强 → 16 强 → 8 强 → 半决赛 → 决赛）推进
4. 重复 10000 次
5. 统计各队：晋级 32 强/16 强/8 强/4 强/决赛/夺冠的频率

### 4.5 投注推荐与赔率偏差检测算法

#### 4.5.1 博彩隐含概率转换

```python
# 十进制赔率 → 隐含概率
P_book(胜) = 1 / odds_win
P_book(平) = 1 / odds_draw
P_book(负) = 1 / odds_lose

# 消除博彩公司 margin（过载率）
total_margin = P_book(胜) + P_book(平) + P_book(负)
P_true(胜) = P_book(胜) / total_margin  # 去 margin 后的"真实"市场概率
```

#### 4.5.2 投注价值判断

核心逻辑：**系统预测概率 > 博彩隐含概率 → 正期望值 → 值得关注**

```
EV(胜) = P_sys(胜) × odds_win - 1
EV(平) = P_sys(平) × odds_draw - 1
EV(负) = P_sys(负) × odds_lose - 1

推荐等级:
  EV ≥ 0.25  → ★★★ 强烈推荐（系统判断显著优于市场）
  EV ≥ 0.15  → ★★  推荐
  EV ≥ 0.05  → ★   值得关注
  EV < 0.05  → 不推荐
```

#### 4.5.3 赔率偏差检测（"特别关注"）

发现系统预测与博彩市场预期的重大分歧：

```
分歧度 Δ = |P_sys(结果) - P_market(结果)|

触发条件（同时满足）：
  1. Δ ≥ 15%                    ← 分歧足够大
  2. 系统置信度 ≥ 60%            ← 系统对自己的判断有把握
  3. 至少有 3 家博彩公司赔率可用   ← 排除单家公司异常

跨平台对比：
  - 多家博彩公司赔率标准差 > 10% → 市场本身存在分歧
  - 最高赔率 / 最低赔率 > 1.5    → 存在明显套利空间
```

#### 4.5.4 置信度指标

系统对自己预测的把握程度（避免低质量比赛的虚假预警）：

```
置信度 = f(Elo 差距稳定性, 数据完整度, 近期状态方差)

- Elo 差距稳定性：两队近期 Elo 波动越小 → 越可信
- 数据完整度：核心球员数据可用比例
- 近期状态方差：状态起伏越小 → 越可信
```

### 4.6 小组赛与淘汰赛区别

- **小组赛**：展示胜/平/负三选项概率及最可能比分
- **淘汰赛**：无平局，加时/点球在模拟层处理，只展示晋级概率

## 五、功能模块

### Tab 1：对战预测（单场）

- 用户选择两支 2026 世界杯参赛球队
- 选择赛制类型（小组赛/淘汰赛）
- 点击预测，展示：
  - **顶层**：圆形百分比（胜/平/负 或 晋级/淘汰）
  - **中层**：雷达图对比 + 各维度柱状图
  - **底层**：可展开的维度分解详情（每个维度得分和贡献百分比）
- **投注建议**（预测结果旁）：
  - 对比 3-5 家博彩公司实时赔率
  - 显示投注价值评分 ★~★★★
  - 标记"市场分歧"场次（博彩公司之间赔率差异大）
- 支持快速切换球队、收藏常用对战组合

### Tab 2：赛事模拟（全局）

- 展示完整 48 队分组 + 淘汰赛对阵图
- 一键"运行模拟"：
  - 蒙特卡洛 10000 次，异步执行
  - 实时展示模拟进度
- 结果展示：
  - 各队晋级概率排行榜（夺冠/决赛/4强/8强/16强）
  - 可点击任意球队查看完整晋级路径概率
  - 淘汰赛对阵图标注各场胜率
- 支持重新模拟、调整模型参数后再次运行

### Tab 3：球队详情

- 48 支球队列表，支持搜索、按洲际筛选
- 球队详情页包含：
  - 基础信息：FIFA 排名、Elo 评分、总身价、世界杯历史战绩
  - 核心球员卡片：首发 11 人预估，每人标注近一年俱乐部胜率、贡献值
  - 世界杯历史战绩时间线
  - 近期国家队比赛战绩列表

### Tab 4：我的

- 收藏的对战预测组合
- 模拟结果历史记录
- 数据更新状态（显示数据时效 + 手动刷新按钮）
- 关于 App、算法说明

### Tab 5：投注推荐

- **今日价值场次**：所有可投注比赛按 EV 排序
  - ★★★ 强烈推荐置顶高亮
  - 每场比赛显示：系统概率 vs 市场概率对比
  - 显示当前最佳赔率及来源博彩公司
- **特别关注**：赔率偏差检测结果列表
  - 系统预测与市场预期分歧 ≥ 15% 的比赛
  - 标注"市场分歧"或"系统看好"标签
  - 跨博彩公司赔率差异可视化（最高/最低/平均）
- **投注历史**（可选功能）：
  - 记录用户的投注和结果
  - 统计推荐准确率

### 7.2 数据同步调度策略

> 考虑到各 API 免费层限额有限，调度策略分为 **比赛日** 和 **非比赛日** 两种模式。

#### The Odds API 调度（限额 500 次/月）

```
比赛日：
  当天比赛  →  每 8 小时更新一次     ← 小组赛每日 3 轮
  明天/后天  →  每天更新 1 次
  其他比赛  →  不更新

非比赛日：
  夺冠赔率   →  每天 2 次（08:00, 20:00）
  ─────────────────────────────────
  月消耗估算:
    主要场次 12天 × 3次/天 = 36 次
    夺冠路赔率 30天 × 2次  = 60 次
    小组赛 8天 × 3次/天    = 24 次
    淘汰赛 7天 × 4次/天    = 28 次
    预留                  ≈ 352 次
    ─────────────────
    总计                    500 次 ✓
```

#### Zafronix 调度（限额 250 次/天）

```
比赛日：
  比赛结果    →  比赛进行中每 15 分钟轮询一次
  积分榜      →  该组比赛结束后更新 1 次

非比赛日：
  不更新（数据不变）
```

```
  非比赛日（每周一）  →  拉取球员俱乐部赛季数据
                        用于更新球员战斗力指数

## 九、异常处理

| 场景 | 处理策略 |
|------|---------|
| 网络不可用 | 离线提示页，展示上次缓存数据，标注时效性 |
| 后端计算超时 | 单场 5s / 模拟 60s 超时，提示重试 |
| 外部 API 拉取失败 | 使用上次成功数据降级，标注"数据可能不是最新" |
| 球队数据不完整 | 缺失维度自动降权，可用维度计算，标注"部分数据缺失" |
| 球员数据缺失 | 新人/边缘球员无数据 → 使用国家队平均水平填充 |
| 模拟任务失败 | 展示失败原因 + 一键重试 |

### 错误码

| 错误码 | 含义 |
|--------|------|
| 1001 | 球队 ID 不存在 |
